📖 Guide Firebase & Synchronisation — Pas à pas ▼ Afficher
🔥 ÉTAPE 1 — Créer un projet Firebase
  1. Allez sur console.firebase.google.com
  2. Cliquez sur "Ajouter un projet" → nom : tb-samayitte-thiadiaye
  3. Désactivez Google Analytics (optionnel) → Créer le projet
  4. Dans le menu gauche : Build → Realtime Database
  5. Cliquez "Créer une base de données" → Région : europe-west1 → Mode test
⚙️ ÉTAPE 2 — Récupérer la configuration
  1. Cliquez sur l'icône ⚙️ (Paramètres du projet) → Paramètres généraux
  2. Descendez à "Vos applications" → Cliquez "</> Web"
  3. Nom de l'app : TB-SamaYitte → Enregistrer
  4. Firebase vous donne un objet firebaseConfig — copiez les valeurs :
const firebaseConfig = {
  apiKey: "AIzaSy...",          ← copiez dans API Key
  authDomain: "projet.firebaseapp.com", ← Auth Domain
  databaseURL: "https://...firebaseio.com", ← Database URL
  projectId: "mon-projet-tb",    ← Project ID
  appId: "1:123:web:abc..."     ← App ID
};
🔒 ÉTAPE 3 — Règles de sécurité (IMPORTANT)

Dans Realtime Database → Règles, remplacez par :

{
  "rules": {
    "sy_thiadiaye_v4": {
      ".read": "auth != null",
      ".write": "auth != null"
    }
  }
}

⚠️ En production, réduire à des règles par rôle utilisateur.

🔑 ÉTAPE 4 — Activer l'Authentification
  1. Menu gauche : Build → Authentication → Commencer
  2. Onglet "Sign-in method" → Activer "Email/Mot de passe"
  3. L'app créera automatiquement les comptes utilisateurs au premier login Firebase
🔌 ÉTAPE 5 — Connecter TB-SamaYitte
  1. Revenez ici → remplissez les champs ci-dessus avec vos valeurs
  2. Cliquez "💾 Enregistrer & Connecter"
  3. Le bouton 🔌 Tester la connexion confirme la synchronisation
  4. Toutes les données (cas, suspects, contacts, DOT) se synchronisent en temps réel
📱 BONUS — Synchronisation KoboToolbox (API REST)

Pour importer automatiquement les données collectées sur le terrain :

  1. Créer un compte sur kf.kobotoolbox.org (gratuit ONG)
  2. Créer un formulaire TB avec les champs : nom_code, commune, village, sexe, age, symptomes, gps_lat, gps_lng
  3. Dans Account Settings → Copier votre token API
  4. Dans TB-SamaYitte → page Structures → entrez votre token + Asset UID du formulaire
  5. Cliquez "🔄 Synchroniser KoboToolbox" → les données arrivent automatiquement
GET https://kf.kobotoolbox.org/api/v2/assets/{ASSET_UID}/data/
Authorization: Token {VOTRE_TOKEN}

🚨 Dépannage — Le CDT / MED ne voit pas les données en temps réel ▼ Afficher
⚠️ Cause n°1 — Firebase non configuré sur l'appareil du CDT

La configuration Firebase doit être saisie sur chaque appareil qui utilise l'application. Si le CDT n'a jamais rempli les champs Firebase sur son téléphone / ordinateur, il travaille en mode local uniquement — il ne reçoit et n'envoie aucune donnée au cloud.

✅ Solution : Sur l'appareil du CDT → Menu Administration → remplir les 5 champs Firebase → cliquer "💾 Enregistrer & Connecter"
⚠️ Cause n°2 — Clé de district différente

Tous les appareils doivent utiliser la même clé district (champ "Clé district" dans la config Firebase). Par défaut : sy_thiadiaye_v4. Si un appareil utilise une clé différente, il lira/écrira dans une autre branche Firebase et ne verra pas les mêmes données.

✅ Vérification rapide — Checklist pour le CDT
☐ Firebase configuré sur mon appareil (icône 🟢 visible en haut)
☐ Clé district identique à celle de l'administrateur (sy_thiadiaye_v4)
☐ Connexion internet active (Wi-Fi ou 4G)
☐ Cliquer le bouton 🔄 en haut pour forcer un rafraîchissement immédiat
☐ Si toujours absent : cliquer "⬇️ Récupérer cloud" dans Administration → Sync
🔌 Tester la synchronisation depuis l'appareil CDT
  1. Ouvrir TB-SamaYitte sur l'appareil CDT
  2. Aller dans Administration → Configuration Firebase
  3. Vérifier que les champs sont remplis → cliquer "🔌 Tester la connexion"
  4. Un message vert "✅ Connexion Firebase réussie" doit apparaître
  5. Revenir au tableau de bord → l'indicateur 🟢 doit être affiché en haut
  6. Demander à l'AC de saisir une donnée test → elle doit apparaître en moins de 3 secondes
📱 Comment partager la config Firebase avec tous les agents ?

L'administrateur (MCD) configure Firebase en premier. Ensuite, il prend une capture d'écran des valeurs ou les note sur papier, puis les partage via WhatsApp avec chaque CDT, ICP, MED pour qu'ils saisissent exactement les mêmes valeurs sur leurs appareils. Une seule configuration Firebase est nécessaire pour tout le district.

📋 Protocole de référence CNDLT — Aide-mémoire
⚡ Actions immédiates (J0)
• Notifier le MCD et le DRSPS
• Isoler le patient (chambre individuelle si possible)
• Constituer le dossier CNDLT (fiche de référence officielle)
• Prélever crachat pour culture et DST
• Bilan VIH obligatoire avant mise sous traitement MR
📋 Dossier de référence CNDLT
• Résultat GeneXpert MTB/RIF (original)
• Fiche de signalement PNLT remplie
• Historique traitement antérieur TB
• Résultats bilan biologique complet
• Radio thoracique (date < 1 mois)
• Statut VIH et traitement ARV si positif
💊 Régime de traitement MR (CNDLT)
• Régime standard OMS : 6 BdqLfxCfzZ + 12 LfxCfzZ
• Durée totale : 18–20 mois minimum
• Suivi culture mensuel (M2, M5, M6, fin)
• DOT obligatoire pendant toute la durée
• Évaluation audiométrique si aminoglycosides
